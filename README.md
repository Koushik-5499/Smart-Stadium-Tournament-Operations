# Volunteer Co-pilot: Smart Stadium & Tournament Operations — FIFA World Cup 2026

A comprehensive, production-grade GenAI-enabled platform designed to enhance stadium operations, safety, and the fan experience for the FIFA World Cup 2026. Built as a unified React application with Firebase and Gemini AI.

## 1. Product Reframe: The Volunteer Co-pilot

**Primary Persona**: A stadium volunteer helping international fans, with no prior operational knowledge and facing language barriers.

**Core User Story**:
A volunteer opens the app during a live match. The app monitors real-time crowd density at nearby gates. When a gate crosses 80% capacity, the app immediately alerts the volunteer with:
1. **Explainable AI Reasoning (XAI)**: A plain-English explanation of *why* the alert fired (e.g., current count, capacity threshold, rate of increase).
2. **Actionable Directive**: Exactly what to do (e.g., redirect incoming fans to Gate X).
3. **Multilingual Assistance**: Auto-translated instructions in the fan's detected/selected language, ready for the volunteer to show or speak.

### Why this depth-first approach?
Following guidance against generic multi-feature apps, this project focuses deeply on ONE persona (the Volunteer) and ONE-TWO verticals (Crowd Management via XAI and Multilingual Assistance). The other modules (Navigation, Control Room, Sustainability, Operations) are integrated as *supporting context* that feeds into the Volunteer Co-pilot, rather than competing for attention as standalone, equal-weight pages.

## 2. Explainable AI (XAI) & Deep Reasoning

- **No hardcoded templates**: Every crowd alert is generated by Google Gemini analyzing live data.
- **Structured Outputs**: Gemini enforces a strict JSON schema for reliability:
  ```json
  {
    "reasoning": "Gate 7 is at 84% capacity and density increased 12% in the last 3 minutes -- redirect recommended...",
    "action": "Redirect fans to Gate 8",
    "severity": "high",
    "translatedInstruction": { "language": "es", "text": "La Puerta 7 está llena..." }
  }
  ```
- **Few-shot Prompting**: System prompts include robust few-shot examples to guarantee format and high-quality, actionable reasoning.

## 3. Algorithmic Optimization

To ensure the application performs optimally at stadium scale, key algorithms have been optimized and their Big-O complexity documented in the code:
- **O(1) Gate Lookup**: Replaced linear O(n) array searches with a Map/dictionary pattern (`getZoneDataFast` in `crowdAnalysis.ts`).
- **Sliding-Window Trend Analysis**: Replaced full historical array recalculations with an O(k) sliding window (where k is the window size) for detecting rising crowd densities (`isRisingTrend` in `crowdAnalysis.ts`).

## 4. Edge Case Testing

Comprehensive unit tests (`crowdAnalysis.test.ts`) have been added to verify extreme and edge scenarios:
- **Off-by-one boundaries**: Exact 80% threshold triggering.
- **Overcapacity**: Handling 0% and 100%+ scenarios gracefully.
- **Missing Data**: Graceful fallback when sensor data is missing.
- **API Failure**: If the Gemini API times out or fails, the system degrades gracefully and provides a rule-based fallback without breaking the UI.
- **Simultaneous Alerts**: Handling multiple gates alarming simultaneously.
- **Malformed Input**: Resilience against unsupported language requests (e.g., requesting 'Klingon').

## 5. Google Services Integration

This project actively leverages several Google services:
- **Google Maps JavaScript API**: Used to render a visual map of the stadium layout directly in the Volunteer Co-pilot dashboard.
- **Firebase Authentication**: Secures access for staff and volunteers.
- **Firebase Firestore**: Stores operational rules, metadata, and simulated state with real-time listeners.
- **Google Gemini API**: Powers the XAI alerts and natural language translations.

## 6. Test with Your Own Data (CSV Upload)

Judges can upload their own CSV data to test the Volunteer Co-pilot's XAI pipeline!
1. Log in to the application.
2. On the Volunteer Co-pilot dashboard, locate the **"Test with your own data"** panel.
3. Upload a CSV file matching the format in `sample-data.csv`.
4. The system will immediately parse the data, override the live feed, and trigger Gemini XAI alerts for any gate exceeding 80% capacity.

## 7. Setup & Local Development

1. **Clone the repository.**
2. **Install dependencies:**
   ```bash
   npm install --legacy-peer-deps
   ```
3. **Configure Environment:**
   Create a `.env.local` file in the root directory:
   ```env
   VITE_GEMINI_API_KEY=your_gemini_api_key_here
   ```
4. **Start Development Server:**
   ```bash
   npm run dev
   ```

## 8. Firebase & Staff Access Setup

1. Install Firebase CLI: `npm install -g firebase-tools`
2. Login: `firebase login`
3. Initialize (if not already): `firebase init`
4. Set Staff Claims via script to access supporting modules:
   ```bash
   node scripts/setStaffClaim.mjs "your.email@example.com"
   ```
